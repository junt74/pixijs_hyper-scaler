# gdd — ゲームとしての仕様（GDD）

**「何をするゲームか」「ルール・判定はどうか」** を記述する場所です。Game Design Document（GDD）や、ゲームロジックの仕様を置きます。

## 記述する内容の例

- **謎解き** … 正解判定ロジック、ヒントの出し方、クリア条件
- **ボードゲーム** … 勝敗条件、ターン進行、特殊ルール、カード効果の定義
- ゲーム全体のコンセプト・ルール概要
- 画面フローやステートの説明（企画・仕様レベル）

技術的な「どう実装するか」は **specs/tech/** に、世界観・ストーリーは **design/** に分けると、AI へのプロンプトが整理しやすくなります。
