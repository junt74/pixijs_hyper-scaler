# ゲーム制作リポジトリ・スタートポイント

ゲーム制作用のリポジトリをゼロから整えるためのテンプレートです。**このリポジトリをクローンして**、新規プロジェクトの土台として使います。

**Cursor** と併用して使うことを想定しています。`.cursor` フォルダを削除すれば、Cursor に依存しない汎用のテンプレートとしても利用できます。

---

## このリポジトリの使い方

### 1. クローンする

```bash
git clone https://github.com/junt74/doc-cursor-gamedev-starter.git
cd doc-cursor-gamedev-starter
```

### 2. 自分のプロジェクト名に合わせてリポジトリを設定する（任意）

- フォルダ名を変更する（例: `my-game-project`）
- リモートを自分の新規リポジトリに差し替える:

  ```bash
  git remote set-url origin https://github.com/<あなたのユーザー名>/<あなたのリポジトリ名>.git
  git push -u origin main
  ```

### 3. 中身をそのまま使う

- **docs/** … ドキュメント・資料用の推奨フォルダ構造が入っています。詳細は [docs/README.md](docs/README.md) を参照してください。
- **.cursor/** … Cursor 用のルールや設定を置く場所です。必要に応じて編集・追加してください。

### 4. ゲームのコードを追加する

- プロジェクトルートに、お使いのエンジンやフレームワークのプロジェクト（`src/`、`package.json`、`project.godot` など）をそのまま配置してかまいません。
- コード用の設定はルートや `src/` に、**説明・仕様・設定資料**は **docs/** に分けておくと整理しやすくなります。

---

## 含まれているもの

| 項目 | 説明 |
|------|------|
| **docs/** | 参照資料・世界観・仕様書・アセットメモ用のフォルダ構造。ゲーム制作向けの配置ガイド付き。 |
| **.cursor/** | Cursor 用のルール（参照ドキュメントなど）。プロジェクトに合わせて編集可能。 |

---

## ドキュメントの整理について

プログラミング以外の資料（エンジン公式ドキュメント、世界観、シナリオ、仕様書など）の置き場所や運用のコツは、**[docs/README.md](docs/README.md)** にまとめてあります。クローン後はそちらを参照してください。

---

## ライセンス・利用について

このスタートポイントは、新規ゲームプロジェクトの土台として自由に利用・改変してかまいません。各プロジェクトのライセンスは、クローン先のリポジトリで設定してください。
